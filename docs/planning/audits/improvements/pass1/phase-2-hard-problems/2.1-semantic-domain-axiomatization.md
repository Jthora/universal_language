# Problem 2.1 — Semantic Domain Axiomatization

**Phase:** 2 (Hard Problems)  
**Location:** New document, likely `foundations/semantic-domain.md`  
**Effort:** 2–4 months  
**Dependencies:** Phase 0 complete  
**Blocks:** 2.2 (algebraic independence requires a domain to test against)

---

## The Problem

The claim "the 11 Σ_UL operations suffice to express ALL finite relationships between distinguishable entities" is **literally undefined** until "all finite relationships" is axiomatized.

Crucially, completeness here does NOT mean "UL contains every possible mathematical structure." It means: every compositional meaning system (first-order logic, type theory, DRT, Montague semantics, natural language) is a Σ_UL-algebra — a specialization/projection of the core. Completeness = every meaning-bearing framework in the target domain receives a unique structure-preserving map from G.

formal-operations.md §3.7 acknowledges this:
> "A full proof would require: 1. Axiomatization of the semantic domain"

No such axiomatization exists anywhere in the project.

## Why This Is Hard

The semantic domain is the set of all "things that can be meant." Defining this formally requires answering: what counts as a meaning structure?

The difficulty is avoiding two failure modes:
1. **Too narrow:** Define the domain as "all finite Σ_UL terms" — then completeness is trivially true by definition, but vacuous. It says nothing about whether UL captures all REAL meanings.
2. **Too broad:** Define the domain as "all possible mathematical structures" — then completeness is trivially false. No finite operation set generates all of mathematics. But this is the WRONG target domain: UL is the minimal skeleton for compositional meaning, not for all of mathematics.

The right formalization must capture: "all compositional meaning relationships that a language might need to express" — without assuming the answer. Existing formal frameworks (first-order logic, lambda calculus, Montague semantics, HPSG, DRT) should all be valid Σ_UL-algebras within this domain.

## Candidate Formalizations

### Option A: Finite Labeled Hypergraphs

**Definition:** A *meaning structure* is a finite labeled hypergraph $H = (V, E, \tau, \lambda)$ where:
- $V$ is a finite set of nodes
- $E$ is a finite set of hyperedges (each connecting a tuple of nodes)
- $\tau: V \to \{E, R, M, A\}$ is a node typing function (each node has a sort)
- $\lambda$ is a labeling function assigning meaning-labels to nodes and edges

**Completeness question:** Can every such hypergraph be constructed from atomic nodes using the 11 Σ_UL operations?

**Pros:** Concrete, graph-theoretically tractable, naturally multi-sorted.
**Cons:** Assumes the 4-sort typing (circular if we're trying to prove 4 sorts suffice). Also, not clear that linguistic meaning HAS hypergraph structure.

### Option B: Relational Structures

**Definition:** A *meaning structure* is a finite relational structure $M = (D, R_1, \ldots, R_k)$ where:
- $D$ is a finite domain (the "entities")  
- Each $R_i$ is a relation of some arity on $D$
- The relations are typed: $R_i \subseteq D^{n_i}$ with sort constraints on each argument position

**Completeness question:** Can every finite relational structure be encoded as a Σ_UL term?

**Pros:** Standard model-theoretic framework. Well-understood. Doesn't presuppose UL structure.
**Cons:** Doesn't naturally capture modification, assertion, or embedding — these are operations, not relations.

### Option C: Term Algebras with Observational Equivalence

**Definition:** The semantic domain is the initial Σ_UL-algebra $T(\Sigma_{UL})$ (all ground terms) quotiented by an observational equivalence: $t_1 \sim t_2$ iff no context $C[\cdot]$ can distinguish them.

**Completeness question:** Is the quotient $T(\Sigma_{UL})/{\sim}$ isomorphic to $G/{\sim_G}$ (the geometric algebra up to indistinguishability)?

**Pros:** Purely algebraic, no external assumptions. If this works, it's the most canonical formalization.
**Cons:** Requires defining "observational equivalence" — what counts as a distinguishing context? This pushes the problem back one step.

### Option D: Generative Definition (Preferred)

**Definition:** A *meaning structure* is anything generated by the free Σ_UL-algebra from a set of atomic meanings. The semantic domain $\mathcal{M}$ is the class of all finitely-generated Σ_UL-algebras.

**Completeness question:** Is $G$ a model that realizes all finitely-generated Σ_UL-algebras? (This is the Adequacy Theorem.) And is this class "rich enough" — does it capture everything a language might need?

**Pros:** Natural, algebraic, well-motivated. Reduces completeness to adequacy + a meta-argument about expressiveness.
**Cons:** The meta-argument ("this class captures all real meanings") is philosophical, not mathematical. But it CAN be supported by showing the class includes: predication, modification, negation, conjunction, disjunction, embedding, abstraction, composition, inversion, quantification — and arguing these are sufficient for compositional semantics.

## Recommended Approach

**Start with Option D (generative), use Option A (hypergraphs) as a sanity check.**

1. Define $\mathcal{M}$ as the class of finitely generated Σ_UL-algebras
2. Prove $G$ realizes all of $\mathcal{M}$ (this is essentially the Adequacy Theorem, already done)
3. Then argue $\mathcal{M}$ is "rich enough" by:
   - Showing it contains all boolean combinations of atomic assertions ✓
   - Showing it contains all quantified assertions ✓ 
   - Showing it contains all nominalized sub-assertions (via embed) ✓
   - Showing it contains iterated modification ✓
   - Comparing with known semantic frameworks (DRT, HPSG, Montague) and checking that their structures are representable
4. If a structure from a known framework is NOT representable, that's evidence for a missing operation (feeds 2.2)

## Deliverable

A document that:
1. Formally defines the semantic domain (which option, precisely stated)
2. Proves that the Adequacy Theorem gives completeness over this domain
3. Lists the expressiveness checks (what CAN be represented)
4. Lists any structures from known frameworks that CANNOT be represented (fueling Phase 2.2)
5. Honestly states the limitations of the chosen formalization

## Validation

- [ ] The phrase "all finite relationships" has a precise mathematical meaning
