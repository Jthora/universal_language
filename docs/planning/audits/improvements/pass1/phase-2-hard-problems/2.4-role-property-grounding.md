# Problem 2.4 — Non-Circular Role Property Grounding

**Phase:** 2 (Hard Problems)  
**Location:** Expansion of `foundations/independent-derivation.md`, potentially new `foundations/grounding-analysis.md`  
**Effort:** 2–4 months  
**Dependencies:** 0.5 (initial role-property derivation attempt)

---

## The Problem

The Unique Grounding Theorem proves: Point↔Existence, Line↔Relation, Angle↔Quality, Curve↔Process, Enclosure↔Concept is the unique mapping preserving 5 role properties.

Phase 0.5 attempts to derive these role properties from 5 independent linguistic traditions without geometric vocabulary. The preliminary analysis (in Task 0.5) suggests:

| Role Property | Independently Derivable? |
|--------------|-------------------------|
| Dependency rank | Yes (from Montague, Langacker) |
| Parameterization | Yes (from Montague, Jackendoff, Langacker) |
| Boundary | Partial (from Jackendoff, Langacker) |
| Symmetry | Weak (Langacker only, informal) |
| Dimensionality | No (purely geometric) |

This raises the critical question: **Does the Unique Grounding Theorem still hold if we drop dimensionality (and perhaps symmetry)?**

## The Mathematical Question

**Reduced Grounding Problem:** Using only {dependency rank, parameterization, boundary} — the three independently motivated role properties — is the mapping Point↔Existence, ..., Enclosure↔Concept still unique?

This is a combinatorial question. There are 5! = 120 possible bijections between {Point, Line, Angle, Curve, Enclosure} and {Existence, Relation, Quality, Process, Concept}. The current proof eliminates all but one using all 5 role properties. How many are eliminated using only 3?

## Approach

### Step 1: Redo the elimination with 3 properties

Construct the 5×3 property matrix (geometric primitives × {dep_rank, parameterization, boundary}):

| Primitive | Dep. Rank | Parameterization | Boundary |
|-----------|-----------|-----------------|----------|
| Point | 0 | No | No |
| Line | 2 (needs 2 points) | No | No |
| Angle | 4 (needs 2 lines) | No | No |
| Curve | varies | Yes | No |
| Enclosure | varies | No | Yes |

And the semantic 5×3 matrix:

| Semantic | Dep. Rank | Parameterization | Boundary |
|----------|-----------|-----------------|----------|
| Existence | 0 | No | No |
| Relation | 2 | No | No |
| Quality | >2 | No | No |
| Process | varies | Yes | No |
| Concept | varies | No | Yes |

**Matching:** A valid mapping must send each geometric primitive to a semantic primitive with the same property values.

- Point (0, No, No) → must map to Existence (0, No, No) ✓ — **forced**
- Line (2, No, No) → must map to Relation (2, No, No) ✓ — **forced** (if Quality has dep rank >2)
- Curve (varies, Yes, No) → must map to Process (varies, Yes, No) ✓ — **forced** (only parameterized one)
- Enclosure (varies, No, Yes) → must map to Concept (varies, No, Yes) ✓ — **forced** (only bounded one)
- Angle (4, No, No) → must map to Quality (>2, No, No) ✓ — **forced** (only remaining option)

**Result: If the property values are as above, the 3-property elimination ALSO gives a unique mapping.**

### Step 2: Verify the property values

The crucial values that need independent verification:

1. **Existence has dependency rank 0.** Can this be derived without geometry?
   - From Montague: type e is basic (not constructed from other types) → rank 0. ✓
   - From Frege: Objects are saturated (not functions requiring arguments) → rank 0. ✓
   
2. **Relation has dependency rank exactly 2.** Can this be derived without geometry?
   - From Montague: type e→(e→t) takes two arguments → rank 2. ✓
   - From Langacker: relations profile connections between trajector and landmark (2 entities). ✓

3. **Process requires parameterization.** Can this be derived without geometry?
   - From Jackendoff: activities are aspectually unbounded, requiring temporal parameterization. ✓
   - From Langacker: sequential scanning tracks a process through time. ✓

4. **Concept requires boundary.** Can this be derived without geometry?
   - From Jackendoff: bounded events (telic) have natural endpoints. For concepts: categories have membership criteria = boundary conditions. Weaker.
   - From Langacker: bounded regions in conceptual space. ✓ (but this IS geometric vocabulary)

5. **Quality has dependency rank >2 and is not parameterized.** Can this be derived?
   - Quality/modification in Montague: type (e→t)→(e→t) — a function on predicates. Takes a predicate as argument, which itself takes entities. Dependency rank ≥ 3? Not exactly — it takes one argument (a predicate), not three entities.
   - This is the weakest point. The claim that Quality has strictly higher dependency rank than Relation is plausible but needs careful formalization.

### Step 3: Assess residual circularity

If the 3-property elimination works and all property values can be independently derived:
→ **The grounding is non-circular.** Upgrade the Unique Grounding Theorem to PROVEN (unconditional).

If some property values REQUIRE geometric vocabulary to state:
→ **The grounding has residual circularity** in those specific properties. Document precisely which.

If the 3-property elimination has ambiguities (two primitives match the same property profile):
→ **The reduced grounding is weaker.** State: "The mapping is unique up to the ambiguity between X and Y." Then assess whether EITHER candidate makes semantic sense.

## The Deeper Issue

Even if the 3 properties suffice for the elimination, there's a meta-question: **Why these properties?**

The 5 role properties were chosen because they distinguish the geometric primitives. That's backwards — the properties should be motivated by what MEANING requires, not by what distinguishes geometric objects.

A truly non-circular grounding would:
1. Start from semantic requirements ("meaning structures need: things that exist, things that connect, things that modify, things that unfold, things that contain")
2. Derive formal properties from those requirements ("things that exist are independent → rank 0")
3. THEN look for geometric objects that satisfy those properties
4. THEN verify uniqueness

Phase 0.5 does step 2 partially. This task completes steps 2–4.
