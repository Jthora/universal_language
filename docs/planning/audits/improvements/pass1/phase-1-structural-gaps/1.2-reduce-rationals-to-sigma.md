# Task 1.2 — Reduce Rational Number Construction to Σ_UL Operations

**Phase:** 1 (Structural Gap Closure)  
**File:** `frontier/expedition-one/numbers-and-computability.md` §2  
**Effort:** 3–5 days  
**Dependencies:** None

## The Gap

numbers-and-computability.md constructs ℚ using Thales' intercept theorem (classical ruler-and-compass geometry). The construction is geometrically correct. But it's not expressed as a sequence of Σ_UL operations.

The ℕ construction IS expressed in Σ_UL terms:
- 0 = origin point (atomic entity)
- n+1 = compose(translate_unit, n) (compose the unit-translation relation n times)

The ℤ construction IS expressed in Σ_UL terms:
- −n = invert(n) (reverse direction of the unit-translation chain)

The ℚ construction is NOT:
- p/q = "the point where the line from 0-to-p intersects the parallel through q" — this is Thales, not Σ_UL.

## Objective

Decompose the Thales construction into Σ_UL operations, or prove it cannot be so decomposed (which would be a significant finding — it would mean ℚ is NOT constructible in UL from the 11 operations alone).

## Approach

The Thales construction for p/q:
1. Start with ℤ-points on a line (the "number line")
2. Construct a second line at angle θ to the first
3. Mark q equal intervals on the second line
4. Connect the q-th mark to p on the number line
5. Draw parallels through the intermediate marks
6. The parallels intersect the number line at multiples of p/q

In Σ_UL terms, each step should decompose as:

| Thales Step | Σ_UL Operations |
|-------------|----------------|
| 1. ℤ-points on line | Already constructed (compose chains) |
| 2. Second line at angle θ | modify_relation(rotation_θ, number_line_segment) — creates a new relation (directed path) at angle θ from origin |
| 3. q equal intervals | compose(unit_on_second_line, ...) — q-fold composition of unit translation along the second line |
| 4. Connect q-th mark to p | predicate(mark_q, connecting_path, point_p) — creates an assertion linking the two points |
| 5. Parallels through intermediate marks | For each mark k (1..q-1): modify_relation(translate_to_k, connecting_path) — translate the connecting path to pass through mark k |
| 6. Intersection with number line | This is the hard step — intersection is not obviously a Σ_UL operation |

## The Hard Part: Intersection

Step 6 requires computing the intersection of two lines. This is a geometric operation, but is it a Σ_UL operation?

**Option A: Intersection as conjoin.** The `conjoin` operation creates the overlap of two constructions. If two lines are assertions (or can be embedded as assertions), their overlap (conjoin) would include their intersection point. But conjoin : a × a → a — it takes assertions, not relations.

**Option B: Intersection as a compound construction.** Intersect(L₁, L₂) = the entity e such that predicate(e, on, L₁) AND predicate(e, on, L₂). This uses: predicate (twice), conjoin (once), and requires existential quantification. Can `quantify` produce the existential? If quantify is ∀ (universal) only, we need ¬∀¬ = ∃ via negate+quantify.

**Option C: Intersection requires a new operation.** If intersection cannot be decomposed, this is evidence that the 11 operations are INCOMPLETE for geometric construction — a significant finding that feeds into Phase 2 Problem 2.1.

## Deliverable

One of:
1. Complete decomposition of Thales' construction into Σ_UL operations (upgrade ℚ to PROVEN)
2. Proof that intersection cannot be decomposed (evidence for incompleteness, feeds Phase 2)
3. Identification of the minimal additional operation needed (e.g., "meet" : r × r → e)
