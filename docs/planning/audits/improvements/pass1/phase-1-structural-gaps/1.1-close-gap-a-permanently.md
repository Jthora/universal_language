## Deliverable

A 1-page formal argument (either added to formal-foundations.md §3.3 or as a lemma in formal-operations.md) that proves:

**Lemma (Structural Distinctness).** Let $e_1, e_2$ be distinct well-formed Σ_UL-expressions over a set of atoms $A$. Let $\phi: A \to G$ be an injective atom-mapping. Then $\hat{\phi}(e_1) \neq \hat{\phi}(e_2)$, where $\hat{\phi}$ is the unique extension of $\phi$ to all expressions.

**Proof sketch:**
1. **Base case:** Distinct atoms have distinct images (by injectivity of $\phi$).
2. **Inductive step:** Let $e_1 = \omega(a_1, \ldots, a_n)$ and $e_2 = \omega'(b_1, \ldots, b_m)$.
   - **Case A:** $\omega \neq \omega'$. Different operations produce geometrically different construction types (e.g., `predicate` produces enclosure+path systems, `modify_entity` produces transformed configurations). The construction types are disjoint subsets of $G_e \cup G_r \cup G_m \cup G_a$. Therefore $\hat{\phi}(e_1) \neq \hat{\phi}(e_2)$.
   - **Case B:** $\omega = \omega'$, but some $a_i \neq b_i$. By induction hypothesis, $\hat{\phi}(a_i) \neq \hat{\phi}(b_i)$. By Operation Distinctness (formal-operations.md §2), $\omega^G$ preserves input distinctness. Therefore $\hat{\phi}(e_1) = \omega^G(\hat{\phi}(a_1), \ldots) \neq \omega^G(\hat{\phi}(b_1), \ldots) = \hat{\phi}(e_2)$.
3. **Conclusion:** $\hat{\phi}$ is injective. ∎

## What Makes Case A Work

Case A requires that different operations produce outputs in non-overlapping geometric "shapes." This needs verification:

| Operation | Output Type | Geometric Character |
|-----------|------------|-------------------|
| predicate | Assertion (enclosure containing path system) | Closed region with internal structure |
| modify_entity | Entity (transformed configuration) | Configuration with transformation marker |
| modify_relation | Relation (transformed path) | Path with transformation marker |
| negate | Assertion (reflected construction) | Mirror image of input |
| conjoin | Assertion (overlapping pair) | Two constructions sharing region |
| disjoin | Assertion (adjacent pair) | Two constructions in disjoint regions |
| embed | Entity (enclosure around assertion) | Enclosure containing assertion-type content |
| abstract | Modifier (from entity) | Transformation derived from entity |
| compose | Relation (chained paths) | Concatenated path |
| invert | Relation (reversed path) | Path with reversed orientation |
| quantify | Assertion (scaled entity) | Scaled configuration |

The key question: can the output of `predicate` ever be structurally identical to the output of, say, `conjoin`? 

If the operations produce outputs with distinguishing structural features (e.g., predicate outputs have exactly one internal path, conjoin outputs have exactly two overlapping assertion-regions), then Case A holds.

This needs to be formally verified — enumerate the structural invariants that distinguish each operation's output type.

## Open Question

If some operations CAN produce structurally identical outputs for different inputs (e.g., `modify_entity(m, e)` could produce the same configuration as some `predicate(e', r, e'')` by coincidence), then Case A fails for those operation pairs and injectivity needs a different argument.

In that case, the fix is: restrict to expressions in **normal form** (a canonical representation where distinct expressions have distinct syntax trees), and prove the Σ_UL operations preserve normal form uniqueness. This is more work but still within reach.
