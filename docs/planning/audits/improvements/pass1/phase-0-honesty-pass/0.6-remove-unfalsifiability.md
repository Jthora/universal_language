# Task 0.6 — Remove Unfalsifiability Shield from paradigm.md

## The Problem

paradigm.md establishes two positions:

1. **Correct position:** UL is a formal mathematical system, not a model of English. Natural languages are not the primary validation target.

2. **Problematic position:** Testing UL against natural languages is a "category error." Human languages are "distorted, incomplete instantiations" of UL. Failures to decompose natural language constructions "prove nothing."

Position 1 is legitimate methodology. Position 2 creates unfalsifiability: if math doesn't prove it (proofs incomplete) AND empirics can't test it (natural languages dismissed), nothing can ever show UL is wrong.

## What to Keep

- The core insight: UL is a formal system and should be validated mathematically, not by checking whether it decomposes English sentences
- The analogy to physics: you don't validate quantum mechanics by checking Aristotelian physics
- The warning against assuming natural languages are ground truth

## What to Change

### Change 1: Soften the epistemic shield

**Current text (paraphrased):**
> UL failing to decompose some English construction proves nothing — maybe that construction is an artifact of English's limitations.

**Replace with:**
> UL failing to decompose some English construction does not automatically invalidate UL — the construction may be an artifact of English's idiosyncratic structure. However, systematic failures across multiple unrelated languages would constitute evidence that UL's 4-sort structure is missing something. Such evidence should be taken seriously, not dismissed.

### Change 2: Add explicit falsification criteria

**New section: "What Would Prove UL Wrong?"**

```markdown
## WHAT WOULD PROVE UL WRONG?

Any mathematical system that claims to be real — in the sense that ℕ is real — 
must be falsifiable within its own domain. The following results would 
constitute evidence against UL's claims:

### Mathematical Falsifiers

1. **A 5th irreducible sort.** If someone exhibits a class of semantic objects 
   that cannot be typed as Entity, Relation, Modifier, or Assertion — and 
   proves this irreducibility with the same rigor as independent-derivation.md 
   §4.1–4.4 — then UL's 4-sort structure is incomplete.

2. **A derivable operation.** If someone shows that one of the 11 Σ_UL 
   operations can be expressed as a term in the other 10, then Σ_UL is 
   not minimal as claimed.

3. **A non-embeddable language.** If someone constructs a Σ_UL-algebra 
   that demonstrably cannot be embedded in G (the geometric realization), 
   then the embedding theorem fails and G is not adequate.

4. **A second valid grounding.** If someone finds a different mapping 
   between geometric and semantic primitives that satisfies the role 
   properties equally well, the "unique" grounding is not unique.

5. **Algebraic equivalence to a smaller system.** If someone proves 
   that the 4-sort structure is algebraically equivalent to a 3-sort 
   structure (one sort is definable from the others, contra the 
   irreducibility proofs), then UL's minimality fails.

### Empirical Evidence That Would Be Informative

While natural languages are not the primary validation target, the 
following empirical observations would be informative (not conclusive):

6. **Systematic embedding failure.** If a class of semantic relationships 
   found across multiple unrelated language families cannot be embedded 
   in Σ_UL without ad hoc extensions, this suggests a gap in the 
   operation set.

7. **Computational intractability.** If parsing or equivalence-checking 
   UL expressions is provably harder than the equivalent operations in 
   existing formal systems (predicate logic, DRT, HPSG), the practical 
   utility claim is undermined even if the theoretical claims hold.

None of these individually would "kill" UL — each would indicate a 
specific revision needed. But if multiple mathematical falsifiers 
held simultaneously, the overall framework would need fundamental 
rethinking.
```

### Change 3: Reframe the paradigm note

**Current:** Appears at the top of multiple documents as a warning that blocks natural-language testing.

**Replace with a more balanced version:**

```markdown
> **⚠ PARADIGM NOTE:** The proofs in this document establish properties 
> of a formal algebraic system. They do not claim to model any specific 
> natural language. Natural language data is informative but not the 
> primary validation target — the validation target is mathematical: 
> minimality, completeness, uniqueness within the formal framework. 
> See `foundations/paradigm.md` for methodology and falsification criteria.
```

This keeps the essential warning but doesn't block empirical evidence from being relevant.
