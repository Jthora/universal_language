# Research 3.5 — Pragmatics

**Phase:** 3 (Extended Theory)  
**Location:** New extension  
**Effort:** 3–6 months  
**Dependencies:** 2.1 (semantic domain), 3.2 (modal logic — epistemic states needed)

---

## The Gap

UL currently operates at the level of SEMANTICS — what sentences mean in isolation. It has no theory of PRAGMATICS — what speakers mean by uttering sentences in context. This omission is enormous:

- **Speech acts:** "Can you pass the salt?" is syntactically a question but pragmatically a request
- **Implicature:** "Some students passed" implicates "not all students passed" (Grice)
- **Presupposition:** "The king of France is bald" presupposes France has a king
- **Deixis:** "I," "here," "now" shift meaning based on who/where/when speaks
- **Context dependence:** "It's cold" means different thresholds in different contexts
- **Discourse structure:** Sentences interact — "He was tired. He went to bed." creates a causal inference

Without pragmatics, UL can represent the propositional content of utterances but not the communicative intentions behind them.

## Framework: Speech Act Theory + Relevance Theory

### Speech acts (Austin, Searle)

Every utterance has three levels:
1. **Locution:** The literal meaning (what was said)
2. **Illocution:** The communicative intention (what was meant)
3. **Perlocution:** The effect on the hearer (what was achieved)

**UL currently captures only locution.** Illocution and perlocution require additional structure.

### Relevance theory (Sperber & Wilson)

Communication succeeds by maximizing relevance: the balance between cognitive effect (new information) and processing cost. 

**Geometric interpretation:** Relevance = the ratio of (semantic distance traveled) to (path complexity). A relevant utterance moves you far in meaning-space via a short path. An irrelevant one uses a long, complex path to move you nowhere.

## Research Questions

### Q1: Can illocutionary force be encoded as a modifier?

**Candidate:** Illocutionary force is a modifier on assertions:
- assert(a) = modify_assertion(declarative, a) — "it is the case that a"
- ask(a) = modify_assertion(interrogative, a) — "is it the case that a?"
- command(a) = modify_assertion(imperative, a) — "make it the case that a!"
- promise(a) = modify_assertion(commissive, a) — "I will make it the case that a"

**Problem:** Same as Task 3.1 — there is no `modify_assertion` in Σ_UL. However, the embedding trick works: `embed(a)` turns an assertion into an entity, which CAN be modified:

$$\text{ask}(a) = \text{predicate}(\text{speaker}, \text{requests\_truth\_of}, \text{modify\_entity}(\text{interrogative}, \text{embed}(a)))$$

This is clunky but expressible. The question is whether it captures the right structure.

### Q2: Can Gricean implicature be derived?

Grice's maxims (quantity, quality, relation, manner) generate implicatures. For example:

**Scalar implicature:** "Some students passed" → not all (because saying "all" would have been more informative; failing to say it implicates its negation).

**Geometric model:** The utterance "some" occupies a region in meaning-space. The utterance "all" occupies a superset region. Choosing the smaller region (when the larger was available) signals that the speaker does not believe the larger region holds.

This requires:
- A model of the speaker's epistemic state (→ Task 3.2, modal logic)
- A preference ordering on utterances (informativeness)
- An inference mechanism (the hearer reasons about why the speaker chose THIS utterance)

### Q3: Can context-dependence be modeled as fiber bundle cross-sections?

From Task 3.3, meaning-space is a fiber bundle with context as base space. A context-dependent expression like "cold" doesn't have a fixed meaning — it has a meaning FUNCTION that depends on context.

Formally: "cold" is a SECTION of the fiber bundle — a function from contexts to meanings.

**This would explain:** Why the same word means different things in different contexts (different fibers), why meaning can be "transported" between contexts (parallel transport), why some context-shifts change meaning and others don't (depends on curvature).

### Q4: Can discourse structure be modeled as path composition?

A discourse is a sequence of assertions: a₁, a₂, ..., aₙ. The discourse as a whole means more than the conjunction a₁ ∧ a₂ ∧ ... ∧ aₙ — there are narrative inferences, causal connections, and rhetorical relations.

**Geometric model:** Each assertion is a segment. The discourse is a path (compose segments). The meaning of the discourse = the path, not just the set of segments.

This would explain:
- Order matters: "He fell. He died." ≠ "He died. He fell."
- Coherence: a good discourse follows a geodesic (shortest path through meaning-space)
- Topic shifts: discontinuities in the path (connects to Task 3.4)

## Approach

### Phase A: Speech acts

1. Define illocutionary force modifiers in Σ_UL (possibly via embed/modify_entity)
2. Show the 5 Searle categories (assertive, directive, commissive, expressive, declarative) can be expressed
3. Verify: the encoding distinguishes performatively different utterances

### Phase B: Implicature

1. Define informativeness ordering on assertion-regions (subset relation + measure)
2. Model scalar implicature as inference about region choice
3. Show: "some" → "not all" follows from the geometric model
4. Assess: does this generalize to other implicature types?

### Phase C: Context and deixis

1. Define context as a point in the fiber bundle base space
2. Define deictic expressions as bundle sections
3. Model context-shift as parallel transport
4. Verify: at least simple examples (indexicals, demonstratives) work correctly

### Phase D: Discourse

1. Represent discourse as composed path (sequence of assertion-segments)
2. Define coherence as a path property (e.g., smoothness, geodesic approximation)
3. Model at least one rhetorical relation (cause, contrast, elaboration) geometrically

## Deliverable

1. Illocutionary force encoding in Σ_UL
2. Implicature derivation mechanism (at least scalar)
3. Context-dependence model via fiber bundles
4. Discourse coherence as path property
