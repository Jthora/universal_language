## The Gap

Natural language is saturated with uncertainty: "probably," "maybe," "most," "unlikely," "roughly." UL currently has no mechanism for expressing graded belief or probabilistic assertions. Everything is either asserted or negated — there is no middle ground.

Gap-analysis.md §1.4 identifies this as a structural limitation: "The current system lacks probabilistic qualification of assertions."

**Framing question:** Is probability already expressible within Σ_UL as a specialization (a Σ_UL-algebra with probabilistic structure), or does it require genuinely new primitives beyond the minimal core? There are three possible outcomes: (1) probability is already a projection of the core — no new operations needed; (2) probability can be encoded via clever composition of existing operations — a derived concept; (3) probability requires new primitives — bounding the minimal core's scope.

## Existing Work

Expedition-two `probability-and-information.md` proves three theorems:

1. **Theorem 1 (Semantic Probability Space):** Meaning-space admits a well-defined probability measure μ over regions, with the sigma-algebra generated by enclosures.
2. **Theorem 2 (KL Divergence as Semantic Distance):** KL divergence between meaning distributions induces a meaningful distance on meaning-space.
3. **Theorem 3 (Fisher Information as Local Curvature):** The Fisher information metric provides local curvature of meaning-space near any distribution.

**Status:** These are exploration-quality proofs. They need to be verified, connected to the core Σ_UL formalism, and their implications worked out.

## Research Questions

### Q1: How does probability connect to the 11 operations?

**Candidate:** Probability is a modifier (sort m). A probabilistic assertion `probably(a)` = `modify_assertion(p_0.8, a)` where p_0.8 is a probability-modifier.

**Problem:** There is no `modify_assertion` operation in Σ_UL. The modification operations target entities and relations, not assertions. Options:
- (a) Add `modify_assertion : m × a → a` to Σ_UL (would change the signature — major revision)
- (b) Encode probability via existing operations: `quantify(prob_m, embed(a))` — treat the probabilistic assertion as a quantified nominalization
- (c) Treat probability as EXTERNAL to Σ_UL — a meta-level annotation, not a first-class operation

Each option has different consequences for the formal system.

### Q2: Does the probability measure respect the algebraic structure?

If $\mu$ is a probability measure on assertion-space:
- Is $\mu(\text{conjoin}(a_1, a_2)) = \mu(a_1) \cdot \mu(a_2)$ when $a_1, a_2$ are independent?
- Is $\mu(\text{negate}(a)) = 1 - \mu(a)$?
- Is $\mu(\text{disjoin}(a_1, a_2)) = \mu(a_1) + \mu(a_2) - \mu(\text{conjoin}(a_1, a_2))$?

These are the standard probability axioms (Kolmogorov). The question is whether the geometric realization (overlap, adjacency, reflection) makes these hold automatically or requires additional axioms.

### Q3: Can hedging operators (probably, possibly, roughly) be encoded?

Natural language hedges include:
- **Probability:** "probably" (p > 0.5), "certainly" (p ≈ 1), "unlikely" (p < 0.3)
- **Approximation:** "roughly," "approximately" (neighbourhood operators)
- **Vagueness:** "somewhat," "fairly" (fuzzy membership)

Can these be cleanly encoded in Σ_UL + probability, or do they require additional machinery (fuzzy logic, supervaluations)?

### Q4: Does the Fisher metric connect to Riemannian geometry (Task 3.3)?

If meaning-space has a Fisher information metric, it's automatically a Riemannian manifold. This would provide a geometric foundation for:
- Semantic distance (geodesic length)
- Semantic curvature (complexity of local meaning variation)
- Semantic volume (information content of a conceptual region)

This connects directly to Task 3.3. The question is whether the Fisher metric from probabilistic considerations and the intrinsic geometry from Task 3.3 agree or conflict.

## Approach

1. Choose between options (a), (b), (c) from Q1 based on mathematical naturality
2. Define the extension rigorously: what new sorts, operations, axioms are needed?
3. Prove: the extension is conservative (doesn't invalidate existing theorems)

### Phase B: Verify expedition-two theorems

1. Check Theorems 1-3 against the rigorous semantic domain (from Task 2.1)
2. Identify any hidden assumptions

### Phase C: Hedging operators

1. Define a small set of probabilistic modifiers {certainly, probably, possibly, unlikely, impossibly}
2. Encode them in the extended signature

1. Compare Fisher metric with any intrinsic metric on meaning-space
2. If compatible: state the compatibility theorem
3. If incompatible: document the conflict and its resolution
