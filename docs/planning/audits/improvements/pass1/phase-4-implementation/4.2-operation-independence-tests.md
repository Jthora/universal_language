# Task 4.2 — Operation Independence Tests

**Phase:** 4 (Implementation Sync)  
**Location:** `crates/ul-core/tests/`  
**Effort:** 2–4 weeks  
**Dependencies:** Phase 2.2 results (algebraic independence analysis)

---

## The Problem

Task 2.2 determines which of the 11 operations are genuinely independent and which (if any) are derivable from others. The implementation needs to reflect this:

- If an operation IS independent: it should have tests demonstrating it does something the others can't
- If an operation IS derivable: it can remain as a convenience function but should be marked as derived, and a test should verify the derivation

## What to Test

### For each independent operation:

Construct a test that demonstrates the operation's behavior cannot be replicated by any composition of the other 10. Concretely:

```rust
#[test]
fn negate_is_independent() {
    // Construct an input where negate produces a result
    // that no composition of the other 10 operations can produce.
    let a = predicate(e1, r, e2);
    let negated = negate(a);
    
    // Show that negated is not equal to any term built from
    // {predicate, modify_entity, modify_relation, conjoin, disjoin,
    //  embed, abstract, compose, invert, quantify} applied to
    // the same inputs.
    assert_ne!(negated, conjoin(a, a));  // not idempotent
    assert_ne!(negated, disjoin(a, a)); // not idempotent
    // ... more witness terms
}
```

This is a FINITE witness test — it doesn't prove independence (that requires the algebra from 2.2) but it demonstrates the operation does something distinct from the others.

### For De Morgan specifically:

If Task 2.2 determines that disjoin is derivable from conjoin + negate (or vice versa):

```rust
#[test]
fn disjoin_equals_de_morgan() {
    // Verify: disjoin(a1, a2) == negate(conjoin(negate(a1), negate(a2)))
    let a1 = predicate(e1, r1, e2);
    let a2 = predicate(e3, r2, e4);
    
    let via_disjoin = disjoin(a1.clone(), a2.clone());
    let via_de_morgan = negate(conjoin(negate(a1), negate(a2)));
    
    assert_eq!(via_disjoin, via_de_morgan);
}
```

### For invert vs. modify_relation:

If Task 2.2 determines that invert IS a special case of modify_relation:

```rust
#[test]
fn invert_equals_modify_relation_reversal() {
    let r = some_relation();
    let m_reverse = reversal_modifier();
    
    let via_invert = invert(r.clone());
    let via_modify = modify_relation(m_reverse, r);
    
    assert_eq!(via_invert, via_modify);
}
```

## Test Categories

### Category A: Property tests (for all 11 operations)

Test algebraic properties that each operation should satisfy:

| Operation | Property | Test |
|-----------|----------|------|
| negate | Involutory | negate(negate(a)) == a |
| conjoin | Commutative | conjoin(a, b) == conjoin(b, a) |
| conjoin | Associative | conjoin(conjoin(a, b), c) == conjoin(a, conjoin(b, c)) |
| disjoin | Commutative | disjoin(a, b) == disjoin(b, a) |
| compose | Associative | compose(compose(r1, r2), r3) == compose(r1, compose(r2, r3)) |
| invert | Involutory | invert(invert(r)) == r |
| embed then abstract | Roundtrip? | abstract(embed(a)) ≈ extract modifier from assertion? |

### Category B: Independence witnesses (for operations confirmed independent by 2.2)

For each independent operation, a test showing it produces outputs unreachable by the other 10.

### Category C: Derivation verifications (for operations found derivable by 2.2)

For each derivable operation, a test confirming the derivation formula.

## Implementation Notes

- Tests should use property-based testing (quickcheck/proptest) where possible
- Each test should reference the specific theorem from Phase 2.2 that justifies it
- Tests for CONJECTURED properties should be `#[ignore]` with a comment: `// Conjectured: see 2.2-algebraic-independence.md`
