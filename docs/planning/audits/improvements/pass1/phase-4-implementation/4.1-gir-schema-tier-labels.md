# Task 4.1 — GIR Schema Tier Labels

**Phase:** 4 (Implementation Sync)  
**Location:** `schemas/` (GIR schema), `crates/ul-core/`  
**Effort:** 1–2 weeks  
**Dependencies:** Phase 0 completion (tier labels must be assigned first)

---

## The Problem

The GIR (Geometric Intermediate Representation) schema currently treats all operations, sorts, and mappings uniformly. There is no distinction between:
- A geometrically forced primitive (PROVEN)
- A structurally constrained mapping (CONDITIONAL)
- A conventional assignment (CONVENTION)

This means the implementation makes the same truth claims as the theory documents — and those claims are being downgraded in Phase 0.

## What Changes

### In the schema:

Add a `tier` field to each definition:

```yaml
# Example: before
operations:
  predicate:
    signature: [entity, relation, entity] -> assertion
    description: "Combine subject, relation, object into statement"

# Example: after
operations:
  predicate:
    signature: [entity, relation, entity] -> assertion
    description: "Combine subject, relation, object into statement"
    tier: T1_geometrically_forced
    proof_status: proven
    proof_ref: "formal-foundations.md §3.2"
```

### Tier values:

- `T1_geometrically_forced` — The operation/mapping is uniquely determined by the geometry. Removing it would break the algebraic structure.
- `T2_structurally_constrained` — The operation/mapping is constrained to a small equivalence class by structural properties. The specific choice within the class is a convention, but the class itself is forced.
- `T3_conventional` — The operation/mapping is one of several viable choices. It's reasonable but not unique.

### Proof status values:

- `proven` — Full proof exists and has been verified
- `conditional` — Proof exists but depends on unverified assumptions (list them)
- `conjectured` — Plausibility argument exists but not a proof
- `open` — No argument exists yet

### In the Rust code:

Add corresponding attributes:

```rust
/// Tier: T1_geometrically_forced
/// Proof: formal-foundations.md §3.2
/// Status: proven
pub fn predicate(subject: Entity, rel: Relation, object: Entity) -> Assertion { ... }

/// Tier: T2_structurally_constrained  
/// Proof: formal-foundations.md §4.3
/// Status: conditional
/// Depends-on: role_property_independence (Task 2.4)
pub fn negate(a: Assertion) -> Assertion { ... }
```
