## The Problem

If Task 3.1 determines that probability integrates with Σ_UL (via a signature extension or an encoding), the implementation needs to support it.

**Do NOT implement until 3.1 is complete.** The design depends entirely on which of the three options is chosen:
- (a) Signature extension: add new operations to ul-core
- (b) Encoding via existing operations: add convenience constructors only
- (c) External annotation: add meta-level probability without changing ul-core

## Implementation Map

### If Option (a) — Signature Extension

New type and operation:

```rust
// New modifier subtype or dedicated type
pub struct Probability(f64); // 0.0 to 1.0

// New operation
pub fn probabilify(p: Probability, a: Assertion) -> Assertion {
    // Attaches probability p to assertion a
    // Returns a "probabilistic assertion"
}
```

GIR schema addition:
```yaml
operations:
  probabilify:
    signature: [probability, assertion] -> assertion
    tier: T3_conventional  # or whatever 3.1 determines
    proof_status: conditional
    depends_on: [probability_extension_theorem]
```

**Impact:** This changes the operation count from 11 to 12. ALL claims about "exactly 11 operations" need to be updated across the project.

### If Option (b) — Encoding

No new operations. Add constructors that compose existing operations:

```rust
/// Constructs a probabilistic assertion using existing Σ_UL operations.
/// Encoding: quantify(prob_modifier(p), embed(a))
/// See: 3.1-probability-uncertainty.md for justification.
pub fn probably(a: Assertion) -> Assertion {
    let prob_mod = probability_modifier(0.7); // conventional threshold
    quantify(prob_mod, embed(a))
}
```

**Impact:** No signature change. The 11 operations remain. Probability is a DERIVED concept.

### If Option (c) — External

No changes to ul-core. Add a wrapper layer:

```rust
pub struct AnnotatedAssertion {
    pub assertion: Assertion,
    pub probability: Option<f64>,
    pub confidence: Option<f64>,
}
```

**Impact:** Cleanest separation. Probability is metadata, not structure.

## Hedging Operators

Regardless of which option is chosen, provide a convenience API for common hedges:

```rust
pub fn certainly(a: Assertion) -> AnnotatedAssertion;   // p ≈ 1.0
pub fn probably(a: Assertion) -> AnnotatedAssertion;     // p > 0.7
pub fn possibly(a: Assertion) -> AnnotatedAssertion;     // p > 0.3
pub fn unlikely(a: Assertion) -> AnnotatedAssertion;     // p < 0.3
pub fn impossibly(a: Assertion) -> AnnotatedAssertion;   // p ≈ 0.0
```

**Note:** The thresholds (0.7, 0.3, etc.) are CONVENTIONAL. Document them as such.
