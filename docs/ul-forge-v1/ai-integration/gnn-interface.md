# Graph Neural Network Interface

> Using GNNs for glyph similarity, analogy completion, and embedding.

---

## Use Case

The GIR is a graph. Graph Neural Networks (GNNs) are the natural ML architecture for learning from graph-structured data. The GNN interface enables:

1. **Glyph similarity** — "Find glyphs structurally similar to this one"
2. **Analogy completion** — "A is to B as C is to ?" (in glyph space)
3. **Glyph embedding** — Map each glyph to a fixed-size vector for downstream tasks
4. **Anomaly detection** — "This glyph has an unusual structure" (possible error)

---

## Architecture

```
GIR JSON → GIR-to-PyG converter → PyTorch Geometric graph → GNN model → embedding vector
```

### GIR → PyG Conversion

Each GIR document becomes a PyTorch Geometric `Data` object:

```python
import torch
from torch_geometric.data import Data

def gir_to_pyg(gir: dict) -> Data:
    # Node features: one-hot encode (type, sort)
    # type: point=0, line=1, angle=2, curve=3, enclosure=4
    # sort: e=0, r=1, m=2, a=3
    node_features = []
    for node in gir['nodes']:
        type_vec = one_hot(NODE_TYPES[node['type']], 5)
        sort_vec = one_hot(SORT_TYPES[node['sort']], 4)
        node_features.append(torch.cat([type_vec, sort_vec]))
    
    x = torch.stack(node_features)  # [num_nodes, 9]
    
    # Edge index: source-target pairs
    # Edge attributes: one-hot encode edge type
    edge_index = []
    edge_attr = []
    for edge in gir['edges']:
        src = node_id_to_idx[edge['source']]
        tgt = node_id_to_idx[edge['target']]
        edge_index.append([src, tgt])
        edge_attr.append(one_hot(EDGE_TYPES[edge['type']], 6))
    
    edge_index = torch.tensor(edge_index).t()  # [2, num_edges]
    edge_attr = torch.stack(edge_attr)          # [num_edges, 6]
    
    return Data(x=x, edge_index=edge_index, edge_attr=edge_attr)
```

### GNN Model

A simple Graph Isomorphism Network (GIN) for glyph embedding:

```python
from torch_geometric.nn import GINConv, global_mean_pool

class GlyphEncoder(torch.nn.Module):
    def __init__(self, in_dim=9, hidden_dim=64, out_dim=32):
        super().__init__()
        self.conv1 = GINConv(nn.Sequential(nn.Linear(in_dim, hidden_dim), nn.ReLU()))
        self.conv2 = GINConv(nn.Sequential(nn.Linear(hidden_dim, hidden_dim), nn.ReLU()))
        self.fc = nn.Linear(hidden_dim, out_dim)
    
    def forward(self, data):
        x, edge_index, batch = data.x, data.edge_index, data.batch
        x = self.conv1(x, edge_index)
        x = self.conv2(x, edge_index)
        x = global_mean_pool(x, batch)  # graph-level embedding
        return self.fc(x)               # [batch_size, 32]
```

## Tasks

### Glyph Similarity

Compute cosine similarity between glyph embeddings:

```python
emb_a = model(gir_to_pyg(glyph_a))
emb_b = model(gir_to_pyg(glyph_b))
similarity = F.cosine_similarity(emb_a, emb_b)
```

Use case: "Show me glyphs in the lexicon that are structurally similar to the one I'm composing."

### Analogy Completion

Using vector arithmetic on embeddings (like word2vec analogies):

```python
# "concept is to structure as process is to ?"
emb_concept = model(gir_to_pyg(concept_glyph))
emb_structure = model(gir_to_pyg(structure_glyph))
emb_process = model(gir_to_pyg(process_glyph))
target_emb = emb_process + (emb_structure - emb_concept)
# Find nearest glyph to target_emb in the lexicon
```

### Anomaly Detection

Train on the 42 canonical glyphs + valid compositions. Flag glyphs with embeddings far from any known cluster.

---

## Training Data

The GNN is trained on:
1. **42 canonical lexicon glyphs** — ground truth
2. **Compositional glyphs** — generated by combining lexicon entries via Σ_UL operations
3. **Augmented glyphs** — canonical glyphs with noise (added/removed nodes/edges) for contrastive learning

Training set is small (hundreds to low thousands of glyphs). GIN can learn from small graph datasets.
